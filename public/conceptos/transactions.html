Transacciones

Una transacción, en el ámbito de las criptomonedas, es una transferencia de valor entre dos carteras o billeteras de un proveedor de criptomonedas que se incluye en el Blockchain. Las carteras de este tipo resguardan un dato secreto llamada llave privada; o bien, el valor que se utilizar para firmar las transacciones y acceder a la cartera, como se mencionó en la sección anterior. Esta llave privada provee una prueba matemática de que el valor provienen del individuo que dice ser el propietario de la cartera.

Al aplicar el algoritmo RSA se obtiene un valor conocido como firma digital, como el que se mencionó anteriormente. Dicha firma establece que un individuo emisor le envía una cantidad determinada de criptomonedas a un usuario receptor y evita que la transacción sea alterada por un tercer individuo, una vez que haya sido emitido. Cabe mencionar que todas las transacciones se transmiten a la red del Blockchain y generalmente comienzan a confirmarse (minarse para obtener el nonce correspondiente) dentro de los 10 - 20 minutos después de haberse llevado a cabo la transacción.

Una transacción es una transferencia de valor de una cartera de criptomoneda a otra. Para que una transacción sea validada y posteriormente, registrada en el Blockchain, es necesario que un minero encuentre el nonce correspondiente para que el hash del bloque en el que se encuentre la transacción cumpla con la regla establecida.

Simulación de una transacción

Instrucciones: A continuación, se le presentan dos campos: firma y verificación. El campo de firma (emisor) le proveerá una llave privada que le permitirá establecer una transacción monetaria (en quetzales). Para ello, puede establecer el nombre del emisor y del receptor, así como la cantidad a transferir. Alternativamente, puede chequear la opción de "Mostrar direcciones" para visualizar las direcciones de los individuos correspondientes. De manera que al generar la firma digital, esta y el mensaje le sea enviado al receptor (campo de verificación). Una vez que sea verificado el mensaje, si este no ha sido modificado, el campo obtendrá un color verde para simbolizar que el proceso de verificación fue un éxito. De lo contrario, se tornará rojo, indicando que la transacción fue alterada.
